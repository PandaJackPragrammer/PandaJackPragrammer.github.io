oracle中进行RMAN备份

Posted on 2025-04-14，7 min read

在生产环境中配置 RMAN（Recovery Manager）备份以确保数据安全，可按以下步骤进行：

1. 环境准备

检查数据库状态

确保数据库处于归档模式，因为归档模式允许在数据库运行时进行备份，且支持基于时间点的恢复。可以通过以下 SQL 语句检查和修改归档模式：

-- 检查归档模式
ARCHIVE LOG LIST;

-- 如果不是归档模式，将数据库切换到归档模式
SHUTDOWN IMMEDIATE;
STARTUP MOUNT;
ALTER DATABASE ARCHIVELOG;
ALTER DATABASE OPEN;

配置快速恢复区（可选但推荐）

快速恢复区（Fast Recovery Area，FRA）是一个集中管理备份和恢复相关文件的区域，可简化备份和恢复操作。可以通过以下命令配置：

--创建恢复区目录
mkdir -p /path/to/fast_recovery_area
-- 如果权限不足，可以使用以下命令修改权限
chown -R oracle:oinstall /path/to/fast_recovery_area
chmod -R 775 /path/to/fast_recovery_area
-- 设置快速恢复区的位置和大小
ALTER SYSTEM SET DB_RECOVERY_FILE_DEST = '/path/to/fast_recovery_area' SCOPE = SPFILE;
ALTER SYSTEM SET DB_RECOVERY_FILE_DEST_SIZE = 10G SCOPE = SPFILE;
-- 重启数据库使配置生效
SHUTDOWN IMMEDIATE;
STARTUP;

2. 连接到 RMAN

使用以下命令以具有适当权限的用户身份连接到 RMAN：

rman target /

3. 制定备份策略

全量备份

全量备份会备份整个数据库，建议定期进行全量备份，例如每周进行一次。在 RMAN 中执行全量备份的命令如下：

BACKUP DATABASE;

增量备份

增量备份只备份自上次备份以来发生变化的数据块，可以减少备份时间和存储空间。分为差异增量备份和累积增量备份：

  • 差异增量备份：备份自上次全量备份或差异增量备份以来所有变化的数据块。例如，每天进行一次差异增量备份：
BACKUP INCREMENTAL LEVEL 1 DATABASE;
  • 累积增量备份：备份自上次全量备份以来所有变化的数据块。例如，每周进行 6 次累积增量备份：
BACKUP INCREMENTAL LEVEL 1 CUMULATIVE DATABASE;

归档日志备份

为了支持基于时间点的恢复，需要定期备份归档日志。可以每天备份一次归档日志：

BACKUP ARCHIVELOG ALL;

4. 自动化备份任务

使用操作系统的任务调度工具（如 Linux 中的 cron）来自动化 RMAN 备份任务。以下是一个每天凌晨 2 点执行全量备份的 cron 示例：

0 2 * * * rman target / <<EOF
BACKUP DATABASE;
BACKUP ARCHIVELOG ALL;
EOF

5. 配置备份保留策略

为了控制备份文件的数量和占用的存储空间，需要配置备份保留策略。例如，设置只保留最近 7 天的备份：

CONFIGURE RETENTION POLICY TO RECOVERY WINDOW OF 7 DAYS;

6. 监控和验证备份

监控备份作业

定期检查 RMAN 备份作业的日志文件，确保备份任务正常完成。可以通过以下命令查看备份作业的详细信息：

LIST BACKUP SUMMARY;

验证备份文件

定期验证备份文件的完整性，确保在需要恢复时能够正常使用。可以使用以下命令验证备份文件：

RESTORE VALIDATE DATABASE;

7. 灾难恢复测试

定期进行灾难恢复测试，以确保在实际发生灾难时能够顺利恢复数据。可以在测试环境中模拟各种故障场景，使用 RMAN 进行恢复操作，验证恢复流程的正确性。

通过以上步骤，可以配置一个完善的 RMAN 备份方案，确保生产环境中的数据安全。

8、以下是一个根据上面配置RMAN备份方法，写一个bash 脚本，确保生产环境中的数据安全，要求每七天进行一次全量备份，每天进行一次增量备份，只保留近30天数据

#!/bin/bash

# 定义 RMAN 命令
RMAN_CMD="rman target /"

# 获取当前日期
DATE=$(date +%Y%m%d)

# 获取当前星期几（0 表示周日，1 - 6 表示周一 - 周六）
DAY_OF_WEEK=$(date +%w)

# 定义备份日志文件
LOG_FILE="/path/to/rman_backup_$DATE.log"

# 定义 RMAN 备份脚本
if [ "$DAY_OF_WEEK" -eq 0 ]; then
    # 周日进行全量备份
    RMAN_SCRIPT="
    CONFIGURE RETENTION POLICY TO RECOVERY WINDOW OF 30 DAYS;
    BACKUP DATABASE PLUS ARCHIVELOG DELETE INPUT;
    CROSSCHECK BACKUP;
    DELETE NOPROMPT OBSOLETE;
    "
else
    # 其他时间进行增量备份
    RMAN_SCRIPT="
    CONFIGURE RETENTION POLICY TO RECOVERY WINDOW OF 30 DAYS;
    BACKUP INCREMENTAL LEVEL 1 DATABASE PLUS ARCHIVELOG DELETE INPUT;
    CROSSCHECK BACKUP;
    DELETE NOPROMPT OBSOLETE;
    "
fi

# 执行 RMAN 备份并记录日志
$RMAN_CMD <<EOF > $LOG_FILE 2>&1
$RMAN_SCRIPT
EOF

# 检查备份是否成功
if [ $? -eq 0 ]; then
    echo "RMAN 备份成功，日志文件: $LOG_FILE"
else
    echo "RMAN 备份失败，日志文件: $LOG_FILE"
fi

以下是一个满足你需求的 Bash 脚本，用于配置 RMAN 备份，实现每七天进行一次全量备份，每天进行一次增量备份，并且只保留近 30 天的数据。

使用说明

  1. 脚本配置：
    • 要把 /path/to/rman_backup_$DATE.log 替换为实际存储备份日志文件的路径。
    • 要保证执行脚本的用户具备执行 RMAN 备份的权限，通常为 oracle 用户。
  2. 权限设置：
    给脚本添加执行权限：
chmod +x rman_backup.sh
  1. 定时任务设置：
    使用 cron 来设置定时任务，让脚本每天定时执行。编辑 cron 表：
crontab -e

在文件里添加以下内容，让脚本在每天凌晨 2 点执行：

0 2 * * * /path/to/rman_backup.sh

保存并退出编辑器，这样脚本就会每天定时执行备份操作，每七天进行一次全量备份，每天进行一次增量备份，同时只保留近 30 天的数据。

脚本解释

  • 全量备份：在周日（DAY_OF_WEEK 为 0）执行全量备份，备份整个数据库和归档日志，并删除已备份的归档日志。
  • 增量备份：在其他时间执行增量备份（差异增量备份），备份自上次备份以来变化的数据块和归档日志，并删除已备份的归档日志。
  • 保留策略：通过 CONFIGURE RETENTION POLICY TO RECOVERY WINDOW OF 30 DAYS 配置保留策略，只保留近 30 天的备份数据。
  • 清理操作：使用 CROSSCHECK BACKUP 检查备份文件的有效性，使用 DELETE NOPROMPT OBSOLETE 删除过期的备份文件。
