Agent 生态大爆发

Posted on 2023-09-14，0 min read

上周 GitHub 上冒出一个叫 Obscura 的项目，短短一个月冲到 1.3 万星

它不是又一个 Agent 框架，也不是又一个模型——它是一个专门为 AI Agent 设计的无头浏览器，用 Rust 重写了浏览器引擎的核心链路。

这篇文章不罗列新闻，就聊透这一个东西：Obscura 到底解决了什么问题，凭什么能火。

Agent 的瓶颈：浏览器太重了

做 AI Agent 的同学应该都遇到过这个问题：想让 Agent 去操作网页、抓数据、填表单，你用什么呢？

市面上无非几个选择：

  • Puppeteer / Playwright — 调 Chrome 的 DevTools 协议，功能强大但启动慢、内存高
  • Selenium — 更重，还要额外装驱动
  • 直接发 HTTP 请求 — 轻倒是轻，但碰上 JS 渲染的页面就直接躺平

这些工具在设计之初想的都是"帮人类做自动化测试"，而不是"给 AI 当浏览器用"。一个 Chrome 实例跑起来内存 200M+，启动要 2 秒，对于需要频繁启停 Agent 的场景来说，太奢侈了。而 Obscura 就是冲着这个痛点来的。

Rust + V8，砍掉一切不需要的东西

Obscura 的架构可以简单理解为：V8 引擎 + Rust 网络栈 + DevTools 协议适配层，砍掉了 Chrome 里所有的 UI 代码、扩展系统、沙箱机制等 Agent 根本用不到的模块。

这套减法做下来效果很震撼：

指标ObscuraHeadless Chrome
内存占用~30 MB200+ MB
二进制体积~70 MB300+ MB
启动时间瞬时~2 秒
页面加载~85 ms~500 ms
Puppeteer 兼容✅✅
Playwright 兼容✅✅

内存降到 1/7，启动几乎是瞬时的。这意味着你可以为每一个 Agent 任务启动一个独立的浏览器实例，用完就销毁，不用操心资源复用和状态污染。

抗检测能力：写给 Agent 的隐身衣

传统的 Headless Chrome 虽然也能工作，但网站可以通过各种特征检测到你正在被脚本操作。Obscura 内置了一套反检测机制：抹掉 webdriver 标记、模拟真实的浏览器指纹、自定义 User-Agent、WebGL/Canvas 指纹模拟。之前要实现这些需要额外挂 puppeteer-extra-plugin-stealth 等插件，现在直接内置了。

实际跑一下

# 下载
curl -LO https://github.com/h4ckf0r0day/obscura/releases/latest/download/obscura-x86_64-linux.tar.gz
tar xzf obscura-x86_64-linux.tar.gz

# 获取页面标题
./obscura fetch https://example.com --eval "document.title"

# 截图
./obscura screenshot https://example.com output.png

如果你已经有基于 Puppeteer/Playwright 的 Agent 代码，只需要把浏览器启动命令替换成 Obscura 就行，API 是兼容的。

一点感想

过去两年 AI 领域大多数爆火的项目都是"给 AI 加脑子"（模型、框架、Agent 编排），但大家慢慢发现脑子再好使，没有手也干不了活。而浏览器就是 AI 最需要的那双手——上网查资料、操作 SaaS 系统、填表单、下订单。

Obscura 踩的正是这个需求：用 1/7 的内存、瞬间启动、内置抗检测，让 Agent 真正能在互联网上干点实事。它不是那种会改变 AI 范式的项目，但它是那种会让很多 Agent 从"玩具"变成"工具"的项目。

想动手试试的，直接去 GitHub 下载就行。
